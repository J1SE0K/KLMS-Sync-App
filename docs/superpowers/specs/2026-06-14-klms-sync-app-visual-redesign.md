# KLMS Sync 앱 시각 재설계 결정

작성일: 2026-06-14

## 결정

KLMS Sync의 새 시각 기준은 다음 조합으로 고정한다.

- 라이트 모드: `P4 Paper Graphite`
- 다크 모드: `D1 Graphite Ivory`

이 조합은 맥, 아이폰, 아이패드, 윈도우 안내 문서에서 같은 제품처럼 보이도록 쓰는 공통 디자인 기준이다. 라이트/다크는 단순 반전하지 않고 각 appearance에 맞는 별도 토큰을 둔다. Apple Human Interface Guidelines의 색상과 Dark Mode 방향도 시스템 appearance에 맞춘 색상 사용을 전제로 하므로, 토큰을 분리하는 방식이 맞다.

참고:

- Apple Human Interface Guidelines - Color: https://developer.apple.com/design/human-interface-guidelines/color
- Apple Human Interface Guidelines - Dark Mode: https://developer.apple.com/design/human-interface-guidelines/dark-mode

## 적용 규칙

`전체 동기화`는 유일한 1차 실행 버튼이다. 라이트에서는 짙은 그래파이트, 다크에서는 아이보리 계열을 사용한다. 이 버튼은 화면에서 가장 먼저 보여야 하며, 실행 삼각형은 같은 텍스트 색을 따른다.

`파일`, `과제/시험`, `공지` 개별 실행 버튼은 포인트 색을 쓰지 않는다. 항상 `secondaryBackground` 기반 중립 버튼으로 둔다. 이 세 버튼은 전체 동기화보다 시각 우선순위가 낮아야 한다.

인증번호, 로그인 완료, 동기화 중단 같은 중요 알림은 실행 영역 바로 위 또는 내부 상단에 배치한다. 인증번호 숫자는 포인트 색을 쓰되, 배경 전체를 포인트 색으로 채우지 않는다.

대시보드 카드, 필터, 정렬, 범위 선택은 박스 형태로 구분한다. 선택 상태는 배경을 아주 약하게 올리고 경계선만 또렷하게 한다. 과한 색면을 쓰지 않는다.

로그와 요청 기록은 별도 로그 탭으로 분리한다. 로그 요약, 동기화 단계, 서버 요청 기록, 파일 요청 기록 순서로 표시한다. 오류와 진단 보기 버튼은 같은 행에서 바로 열 수 있어야 한다.

## 기기별 레이아웃

### Mac

Mac은 작업 관제형 화면을 유지한다.

- 상단: 중요 알림, 전체 동기화, 개별 실행 버튼
- 중단: 대시보드 카드와 선택한 카드 상세
- 하단 또는 별도 탭: 로그 요약, 요청 기록, 진단
- 설정: 별도 창

창 크기에 따라 대시보드 카드는 유동적으로 줄바꿈한다. 목록이 많은 파일/공지 화면은 전체 화면 스크롤 흐름을 유지하고, 내부 일부 영역만 따로 스크롤되지 않게 한다.

### iPhone

iPhone은 한 손 조작 중심이다.

- 상단: 중요 알림과 전체 동기화
- 바로 아래: 파일, 과제/시험, 공지 개별 실행
- 대시보드 카드를 누르면 같은 화면 아래에 상세가 펼쳐진다.
- 파일/공지처럼 원소가 많은 목록은 lazy rendering을 전제로 한다.
- 정렬, 필터, 범위 선택은 박스형 컨트롤로 분리한다.

### iPad

iPad는 넓은 화면을 사용한다.

- 사이드바: 대시보드, 파일, 공지, 과제/시험, 캘린더, 로그, 설정
- 가운데: 목록 또는 카드 그리드
- 오른쪽: 선택한 항목 상세
- 대시보드 동기화 패널: `전체 동기화`를 가장 크게 두고, 바로 아래에 `파일`, `과제/시험`, `공지` 개별 실행 버튼을 3열로 둔다.

아이패드는 iPhone 화면을 늘려 쓰지 않는다. `NavigationSplitView` 성격의 2-3열 구조를 기본으로 둔다.

### Windows

Windows 안내 문서는 Mac/iPhone과 같은 토큰과 정보 구조를 따른다. WinUI 구현 시에도 `전체 동기화`만 1차 색을 쓰고, 나머지 실행/필터/선택 컨트롤은 중립 회색 계열로 유지한다.

## 구현 시 주의점

색상 토큰은 SwiftUI 코드에 흩뿌리지 말고 공유 theme 구조로 묶는다. Mac과 iOS/iPadOS가 같은 토큰 이름을 쓰고, Windows 문서도 같은 이름을 따른다.

라이트/다크 모드는 단순 색상 반전이 아니다. 각 appearance의 토큰을 따로 정의하고, 시스템 appearance 전환에 따라 자동으로 바뀌게 한다.

접근성 기준으로 주요 텍스트와 버튼 텍스트의 대비를 유지한다. 다크 모드의 `primaryAction` 위에는 반드시 `primaryActionText`를 사용하고, 흰색 텍스트를 얹지 않는다.

기존 기능 동작은 이 디자인 변경의 범위가 아니다. 공지 메모, 파일 동기화, 서버 요청, 캘린더/미리알림 로직은 그대로 유지하고, 화면 구조와 스타일만 바꾼다.

## 검증 기준

- Mac, iPhone, iPad에서 같은 팔레트로 보인다.
- 라이트 모드는 `P4 Paper Graphite`, 다크 모드는 `D1 Graphite Ivory`로 보인다.
- `전체 동기화`만 1차 색으로 강조된다.
- 개별 실행 버튼은 중립 회색 계열이다.
- 대시보드 항목을 누르면 상세가 같은 흐름 안에서 보인다.
- 로그/요청 기록은 별도 로그 탭에서 확인된다.
- iPhone의 많은 목록에서 터치 지연이 눈에 띄지 않는다.
- iPad는 사이드바와 상세 패널을 활용한다.
