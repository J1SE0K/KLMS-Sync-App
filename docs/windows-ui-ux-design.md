# KLMS Sync Windows UI/UX 기준

이 문서는 Windows companion 앱을 Mac/iPhone 앱과 같은 제품처럼 보이게 만들기 위한 기준이다. 실제 서버 URL, 토큰, 개인 경로, KLMS 원본 URL은 문서나 코드 기본값에 넣지 않는다.

## 제품 톤

- 조용한 작업 도구처럼 보여야 한다. 마케팅 화면, 큰 히어로, 장식용 그래디언트는 쓰지 않는다.
- 첫 화면은 바로 상태와 실행이 보여야 한다. 설명 페이지를 첫 화면으로 두지 않는다.
- Mac/iPhone과 같은 말투를 쓴다. 예: `대시보드`, `동기화`, `작업 공간`, `연동 상태`, `로그 요약`, `공지 메모도 업데이트`.
- 오류는 크게, 원인은 짧게, 다음 행동은 버튼으로 보여준다.

## 색상 토큰

Windows도 Mac/iPhone/iPad와 같은 `P4 Paper Graphite` 라이트 모드, `D1 Graphite Ivory` 다크 모드 기준을 따른다. 최신 시각 결정 문서는 [2026-06-14-klms-sync-app-visual-redesign.md](./superpowers/specs/2026-06-14-klms-sync-app-visual-redesign.md)이다.

| 토큰 | Light | Dark | 용도 |
| --- | --- | --- | --- |
| `app-bg` | `#F8F7F2` | `#10100F` | 전체 배경 |
| `card-bg` | `#FFFFFF` | `#1D1D1B` | 카드/패널 |
| `subtle-bg` | `#ECE9DF` | `#2D2B27` | 사이드바, 보조 패널, 개별 실행 버튼 |
| `border` | `#D7D1C4` | `rgba(255,255,255,0.105)` | 일반 구분선 |
| `text` | `#171613` | `#F7F3EA` | 주요 텍스트 |
| `muted` | `#6D675D` | `#BDB5A7` | 보조 텍스트 |
| `primary-action` | `#2A2A27` | `#F0DFB8` | `전체 동기화` 기본 실행 버튼 |
| `primary-action-ink` | `#FFFAF0` | `#15130E` | `전체 동기화` 버튼 위 글자/아이콘 |

`전체 동기화` 기본 실행 버튼에만 `primary-action`을 쓴다. 라이트 모드는 짙은 그래파이트, 다크 모드는 아이보리 계열을 써서 두 모드가 같은 제품처럼 이어지게 한다. 아래의 `파일`, `과제/시험`, `공지` 개별 실행 버튼은 `subtle-bg` 기반 중립 버튼으로 둔다. 전체 동기화 버튼 오른쪽의 실행 삼각형은 `primary-action-ink`를 따른다. 선택된 탭, 진행 중 상태, 일반 버튼, 카드/패널/내비게이션 배경은 중립 surface를 유지한다. 파란색은 파일 종류나 링크 보조 정보처럼 의미가 분명한 곳에만 제한적으로 쓴다.

## 화면 구조

Windows는 넓은 화면을 활용해 Mac과 같은 작업대형 구조를 기본으로 한다.

- 상단 중요 배너: 인증번호, 인증 완료, 실행 중, 진단 필요 상태
- 좌측 작업 레일: `동기화`, `작업 공간`, `연동 상태`
- 작업 공간 내비게이션: `대시보드`, `로그`, `진단`
- 우측 콘텐츠: 대시보드 카드, 선택 카드 상세, 로그/진단 패널
- 로그 화면: `로그 요약`, 동기화 단계, 원격 실행 기록, 서버 요청 기록, 파일 요청 기록, 공유 실행 로그를 한곳에 모음

좁은 창에서는 좌측 내비게이션을 상단 탭 또는 하단 탭으로 접는다. 선택된 탭은 회색 계열 선택 배경을 사용한다.

## 주요 화면

### 대시보드

- 대시보드 카드 순서는 `파일`, `과제`, `공지`, `시험`, `헬프데스크`다.
- 과제, 시험, 공지, 파일 카드는 0개인 항목을 숨긴다.
- 메일 내용 자동 판독에서 대시보드에 반영한 과제, 시험, 공지, 파일 항목도 같은 카운트에 포함한다.
- 메일 판독으로 등록한 시험도 별도 “메일 분석” 그룹으로 빼지 말고 기존 시험 카드와 시험 목록에 합친다.
- 캘린더 진단에는 `KLMS 시험`, `메일 등록 시험`, `표시 시험 합계`를 구분해서 보여준다. 사용자에게는 합계가 먼저 보이고, 상세 설명에서 출처를 풀어준다.
- 카드 클릭 시 화면 이동보다 우선으로 카드 아래에 인라인 상세 목록을 펼친다.
- 상세 보기 버튼을 누를 때만 별도 상세 화면으로 이동한다.

### 실행

- `메일·캘린더 분석`은 실행 영역 맨 위에 작게 접힘 패널로 둔다.
- `전체 동기화`는 그 아래에 가장 크게 둔다.
- `파일`, `과제/시험`, `공지`는 그 아래 3분할 짧은 버튼으로 둔다.
- `공지 메모도 업데이트`는 실행 패널 안의 토글로 둔다.
- `미리보기`와 `점검 도구`는 기본 접힘 상태로 둔다.
- 실행 중에는 `중단` 버튼을 같은 패널 안에서 노출한다.

### 서버 연결

- 서버 URL과 클라이언트 토큰 입력/붙여넣기/복사 기능을 제공한다.
- 로컬 IP, 집 주소, 사설 주소처럼 개인정보성 연결 주소는 복사 대상에서 제외한다.
- 연결 확인 성공/실패는 toast와 카드 상태 둘 다에 표시한다.

### 파일

- 정렬 기준: `최신 등록순`, `과목순`, `파일명순`, `상태순`
- 필터: `새 파일`, `첨부 파일`, `강의자료`, `숨김`, `격리`
- 파일 클릭 시 먼저 인라인 상세를 열고, `열기 링크 요청`을 별도 버튼으로 둔다.
- 링크가 준비되면 만료 시간과 남은 열람/다운로드 횟수를 보여준다.

### 공지

- 필터: `새 공지`, `읽지 않음`, `중요`, `숨김`, `과목별`
- 읽음/중요은 각각 토글 버튼으로 둔다.
- 공지 안에서 검출된 시험/과제 후보는 같은 상세 패널에서 보여준다.

### 로그

- 최근 요청, 파일 요청, 설정 변경 요청을 분리해서 보여준다.
- 긴 원본 로그는 기본 접힘 상태다.
- 로그 요약은 항상 먼저 보인다.
- 인증번호, 인증 완료, 파일 요청 실패 같은 중요 이벤트는 상단 알림 영역에도 남긴다.

### 메일·캘린더 분석

- 메일 본문 입력은 앱 안에서만 분석하고 서버에는 원문을 보내지 않는다.
- 결과 카드는 `과제 후보`, `시험 후보`, `공지 후보`, `파일 후보`, `미분류`로 구분한다.
- `등록`, `수정`, `제거` 버튼은 과제/시험/공지/파일 카운트와 상세 목록을 즉시 갱신한다.
- `수정`은 기존 메일 항목을 같은 ID로 덮어써서 중복 카운트를 만들지 않는다.
- 서버가 연결되어 있으면 등록/수정은 `mailDashboardAdd`, 제거는 `mailDashboardRemove` 요청 상태를 로그 화면에도 남긴다.
- 캘린더 등록은 별도 액션으로 두고, 대시보드 반영과 역할을 섞지 않는다.
- 캘린더 상세 항목은 `등록`, `수정`, `삭제` 버튼을 나란히 제공한다. `열기`는 보조 확인 기능이고 처리 버튼을 대체하지 않는다.
- 캘린더 검사에서 `manual_exam_count`가 있으면 오류처럼 보이지 않게 설명한다. 예: `KLMS 시험 2개 · 메일 등록 시험 1개 · 표시 3개`.

## 컴포넌트 기준

- 카드 radius는 8px.
- 카드 안에 또 큰 카드를 넣지 않는다. 반복 항목만 작은 카드로 쓴다.
- 버튼은 아이콘 + 짧은 텍스트를 기본으로 한다.
- 익숙한 동작은 아이콘을 쓴다: 새로고침, 중단, 복사, 열기, 다운로드, 숨김, 복구.
- 각 버튼에는 hover tooltip을 둔다.
- 목록 row는 제목, 과목/학기, 상태 chip, 최근 변경 시각 순서로 배치한다.
- 파일/공지처럼 항목이 많은 목록은 처음부터 전체 DOM을 만들지 않는다. 초기 120개 정도만 렌더링하고, `더 보기` 행으로 120개씩 추가한다.

## 반응성

- 버튼을 누르면 즉시 optimistic 상태를 표시한다. 예: `요청 전송됨`, `Mac 처리 대기 중`.
- 클릭 피드백은 색상, 밝기, 테두리 변화로 처리한다. 목록 행이나 탭에서 scale 애니메이션, 큰 그림자, 긴 transition을 쓰지 않는다.
- 선택 상태는 배경과 1px 경계선으로만 또렷하게 표시한다. 항목 수가 많은 파일/공지 목록에서 shadow를 반복 적용하지 않는다.
- 서버 polling 중이어도 수동 새로고침은 무시하지 않는다. 진행 중이면 다음 갱신으로 예약한다.
- 실시간 이벤트를 받았을 때는 전체 대시보드 데이터를 매번 다시 읽지 않는다. `state`, `sync-data`, `file-access:*`, `shared-settings`, `logs-display:*` reason별로 필요한 패널만 부분 갱신한다.
- 원격 명령/항목 처리/파일 링크 요청 직후에는 전체 `sync-data` 2000개를 다시 읽지 않고 해당 요청 목록만 먼저 갱신한다.
- 진행 중 요청은 1초 이하 체감 간격으로 상태가 바뀌어야 한다.
- 긴 파일 업로드/다운로드는 시간 제한보다 완료 이벤트 기준으로 상태를 갱신한다.
- 검색 입력은 짧게 debounce해서 타이핑마다 전체 목록을 다시 만들지 않는다. 정렬/카드 전환 시에는 표시 개수를 초기값으로 돌려 첫 반응을 빠르게 유지한다.

## 보안/개인정보

- 토큰은 Windows Credential Manager 또는 Electron safeStorage에 저장한다.
- 안전 저장소가 불가능하면 저장하지 않고 이번 세션에서만 사용한다.
- 클립보드로 복사한 토큰은 60초 뒤 자동 삭제한다.
- 로그에 토큰, 서버 secret, 로컬 집 주소, KLMS raw URL, 개인 파일 경로를 남기지 않는다.
- GitHub에는 사용 방법과 placeholder만 올린다.

## 구현 체크리스트

- Mac/iPhone/iPad와 같은 `primary-action` 색을 `전체 동기화` 버튼에만 적용
- `파일`, `과제/시험`, `공지` 개별 실행 버튼은 회색 계열인지 확인
- 상단 중요 배너가 인증번호/실행 중/진단 필요를 먼저 보여주는지 확인
- 좌측 작업 레일이 `동기화`, `작업 공간`, `연동 상태` 순서인지 확인
- 작업 공간 내비게이션이 `대시보드`, `로그`, `진단` 중심인지 확인
- 선택된 탭과 일반 버튼은 회색 계열인지 확인
- 실행 패널과 내비게이션 배경이 포인트색으로 과하게 깔리지 않는지 확인
- 대시보드 카드 0개 항목 숨김
- 카드 클릭 시 인라인 상세 표시
- 미리보기/점검 도구 기본 접힘
- 서버 연결 정보 붙여넣기/복사/검증
- 파일 요청 상태와 링크 만료 표시
- 큰 파일/공지 목록은 초기 120개만 렌더링하고 `더 보기`로 점진 표시
- 로그 요약 + 원본 로그 접힘
- 토큰 안전 저장 실패 시 평문 저장 거부
